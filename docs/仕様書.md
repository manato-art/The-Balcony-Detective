# ベランダ探偵 — 仕様書

> マンションのベランダ・洗濯物・生活感だけを見て「誰が住んでいるか」を当てる、飲み会向けスマホ推理ゲーム。
> 本書は **実装済みの現状（as-built）を記述した仕様書** です。コード優先で作られてきたため、本書は最新コード（`main` ブランチ）の動作を正とします。

- リポジトリ: `manato-art/The-Balcony-Detective`
- 本体: `~/Desktop/veranda-tantei`（依存ゼロの静的Webアプリ。`index.html` を直接開いても、`node server.js` でも動作）
- 想定端末: スマートフォン縦画面（1台を回して複数人で遊ぶ「パスアラウンド」方式）
- 対象: 大学生〜の飲み会。ソフトドリンク参加OK・飲酒強要は禁止（タイトルに明記）

---

## 1. コンセプト

「この部屋、誰が住んでると思う？」を全員で当てる推理ゲーム。洗濯物・生活感は覗き見ではなく **推理のヒント**。黒レース・プロテイン・缶チューハイ・子供服などの生活感から住人を推測する。外した人・自信満々で外した人・一番偏見がキモかった人が飲む。

MVP＝通常クイズモード（本書の対象）。将来的に人狼モードを予定（§14）。

---

## 2. 用語

| 用語 | 意味 |
|------|------|
| ターン | 1人のプレイヤーが1部屋を推理する単位 |
| 張り込み | 対象部屋のベランダを観察している状態 |
| 追加調査 | ヒントを増やす任意アクション（各ターン各1回） |
| ハプニング/カットイン | 住人帰宅など、画面を占有する演出イベント |
| 容疑者リスト | 回答の4択（住人アバター＋名前） |
| 図鑑 | 出会った住人・見つけたアイテムのコレクション |
| シークレット | 低確率でのみ登場するレア住人・レアアイテム |

## 3. 画面遷移

```
タイトル ──┬─→ プレイヤー設定 ─→（マンションはランダム決定）─→ ターン開始（建物全景→ズーム）
           │                                                          │
           ├─→ 図鑑（人物/アイテムタブ）                              ▼
           ├─→ 遊び方（図解7ステップ）                        捜査画面（ベランダ＋追加調査）
           ├─→ 飲みルール                                             │
           └─→ ゲーム設定（歯車）                                     ▼
                                                              回答（4択）→ 結果発表
                                                                     │
                                            全ターン終了 ─────────────┴─→ 最終結果（ランキング＋審判）
```

- 常設UI: 右上に **サウンドON/OFF**、タイトル左上に **ゲーム設定（歯車）**。
- ターン開始時、写真的な建物全景SVGに対象部屋がハイライト→タップでズームインして潜入。

## 4. ゲーム進行ルール

### 4.1 セットアップ
- プレイヤー1〜8人（初期値「プレイヤー1」「プレイヤー2」が入力済み。タップで全選択→上書き）
- 1人あたりの回数 1〜5回
- マンションは10種から **ランダム自動決定**（選択画面なし）
- 総ターン数 = プレイヤー人数 × 回数。登録順（＝着席順）で回す

### 4.2 1ターンの流れ
1. パス画面でスマホを次のプレイヤーへ。対象部屋（例: 402号室）が提示される
2. 「張り込み開始」で建物全景から対象部屋へズームイン
3. **住人帰宅ハプニング判定**（§8。発生時はここでカットイン→ターン終了）
4. 捜査画面: ベランダのヒント4個が初期表示。追加調査でヒントを増やせる
5. 容疑者4択から回答（自信満々宣言は任意）
6. 結果発表（正解/不正解＋煽り文＋飲み指示＋得点）
7. 次のプレイヤーへ。全ターン終了で最終結果

---

## 5. 得点・コンボ

| 項目 | 内容 |
|------|------|
| 正解 | +100点 |
| 自信満々ボーナス | 正解かつ「自信満々」宣言で +50点 |
| ノー調査ボーナス | 追加調査を1回も使わず正解で +50点 |
| コンボ | 連続正解 n 回目に `+20 × (n-1)`（上限+100） |
| 不正解 | 0点。コンボはリセット |

最終結果ではスコア順にランキング。同点は飲んだ口数が少ない方が上位。

## 6. 追加調査（各ターン各1回）

| アクション | 効果 | リスク |
|-----------|------|--------|
| **観察** | 隠れヒントを最大2個開示 | なし（安全） |
| **ポスト** | 強ヒント（決定的情報）を開示 | 既定 **50%** で管理人に発覚（設定変更可・カメラ作動後は+30%、上限95%）。発覚で一口＋以降は回答のみ |
| **待つ** | ランダムイベント（§7） | 帰宅・雨・カメラ・発覚などが起こりうる |
| **聞き込み** | 隣人の噂 | **65%** 本物 / **35%** 嘘（他の容疑者の噂を提示） |

- ヒントは全9個。初期表示4個＋追加調査プール5個（＋強ヒント2個）。
- 発覚後・帰宅後は追加調査ロック、回答のみ可能。

## 7. ランダムイベント（「待つ」／全12種）

同一ゲーム内では全種を消化するまで重複しない（重み付き非復元抽選）。

| id | 種別 | 効果 | 重み |
|----|------|------|------|
| curtain | ヒント | カーテンが開き部屋内が見える | 10 |
| light | ヒント | 電気が点く | 8 |
| mekure | ヒント | 洗濯物がめくれて奥が見える | 8 |
| takuhai | 強ヒント | 宅配ラベルをチラ見 | 8 |
| neighbor | 噂 | 隣人が出てきて本物の噂をタダ聞き | 9 |
| crow | 空振り | カラスが横切る | 8 |
| cat | 空振り | 猫が横切る | 6 |
| tv | 空振り | 隣のテレビの音のみ | 6 |
| camera | 警告 | 防犯カメラ作動→以降ポスト発覚率+30% | 8 |
| rain | 警告 | 雨で洗濯物取り込み→以降の洗濯物ヒント打ち止め | 7 |
| kanrinin | 発覚 | 管理人巡回。見つかって一口＋回答のみ | 12 |
| kitaku | タイマー | 住人帰宅→10秒カウントダウン回答 | 10 |

イベントは可能な範囲で **絵で表現**（カーテンが実際に開く／窓に電気／雨粒＋暗転／カラス・猫が横断／窓に人影）。

## 8. 住人帰宅ハプニング（カットイン）

- **発生タイミング**: 張り込み開始直後（ズーム後）。既定確率 **18%**（設定変更可）
- 演出: パチンコ風の斜めストライプ＋白フラッシュ＋驚愕マスコットで「住人帰宅！！」がスラムイン。緊迫BGMに自動切替
- **飲む対象**（ランダム）: 本人40% / 右隣20% / 左隣20% / 本人以外全員20%。隣は登録順
- ボタン「グイッ してから逃走」を押すまで表示継続 → 押すとそのターンは回答なしで終了、次のプレイヤーへ
- 「待つ」経由の帰宅（§7 kitaku）は10秒タイマー回答（別系統）。二重帰宅は防止

---

## 9. 管理人発覚

- ポスト調査や「待つ」で管理人に見つかると、紺ストライプのカットイン（管理人アバター登場・「見つかった！！」）
- 「グイッ してから回答する」→ **ベランダ＋容疑者リストだけの最終回答ページ**へ。容疑者タップで即回答確定
- 一口は「見つかった分」として結果画面で精算

---

## 10. 飲みルール（表示・口頭運用）

| 状況 | 飲む |
|------|------|
| 不正解 | 一口 |
| 正解 | セーフ |
| 管理人に見つかる | 一口 |
| 自信満々で外す | 二口 |
| 全員が同じ答えで外す | 全員一口 |
| 少数派で正解 | 他全員一口 |
| 一番偏見がキモかった人 | 一口 |

- 「全員同じ」「少数派」「偏見王」は口頭運用。最終結果で全員が同時に偏見王を指差す審判を促す。
- **飲酒強要禁止・20歳未満/ソフトドリンク参加OK** をタイトルに明記。

## 11. データ仕様

### 11.1 住人（全50種）
`js/data/residents.js`

| カテゴリ | 数 | 例 |
|----------|----|----|
| そのへん枠 (normal) | 12 | 女子大生・会社員・主婦・フリーター・在宅ワーカー・新婚夫婦・留学生 ほか |
| 盛り上がり枠 (hot) | 14 | キャバ嬢・ホスト・港区女子・筋肉兄貴・ガチオタク・ゲーマー配信者・ヨガ ほか |
| 罠枠 (trap) | 12 | 管理人・警察官・コスプレイヤー・猫屋敷の住人・単身赴任のパパ・住んでない部屋 ほか |
| レア枠 (rare) | 10 | 元アイドル・占い師・霊媒師・天才投資家・大食いYouTuber・覆面作家 ほか |
| シークレット (secret) | 2 | ハト・宇宙人（§12） |

各住人フィールド: `id, name, cat, icon, desc, hints[9], strong[2], rumor[], roastWrong[], roastRight[], confuse[3]`。
- `hints`/`strong`: `[表示テキスト, アイテム種別]`
- `confuse`: 混同しやすい住人id（4択の誤答候補に優先採用）
- シークレットは誤答候補に混ざらない（正解の時だけ登場）

### 11.2 マンション（全10種）
`js/data/mansions.js`。各20部屋（101–105 / 201–205 / 301–305 / 401–405）。

ボロアパート／学生マンション／繁華街ワンルーム／港区タワマン／郊外ファミリーマンション／デザイナーズマンション／社宅・団地／高級低層マンション／怪しい雑居ビル風マンション／リゾート風マンション

各マンションは住人の出現重み（`pool`）を持ち、生息地が偏る（例: 繁華街＝キャバ嬢/ホスト多め、学生マンション＝留学生/ゲーマー）。

### 11.3 アイテムイラスト
`js/scene.js`。ヒント文のキーワードから **特化イラスト87種** に自動振り分け（`classify`）。該当しない箱は見た目3種に分散し「同じ箱が並ばない」。基本形状21種。
- 例: 黒レース→フリル下着 / シャンパン→木箱にボトル / 盆栽 / 浮き輪 / 盛り塩 / 缶タワー / ピザ箱 / 電球 …
- アイテムはベランダの吊り3＋前列5＋後列4＝最大12枠に配置。タップで名称ツールチップ。

### 11.4 4択生成ロジック
正解＋`confuse`優先＋同カテゴリ＋全体の順で重複なく4件（`CHOICE_N=4`）。

## 12. 図鑑・シークレット

- タイトル「図鑑」→上タブで **人物 / アイテム** 切替（`js/dex.js`、`localStorage: vt_dex_v1`）
- 未発見はシルエット＋「？？？」、発見で解放。人物は遭遇回数も表示
- 解放条件: 人物＝正解発表で明示された時、アイテム＝ベランダに出現した時
- **シークレット住人**（既定3%で通常抽選を乗っ取り）: ハト／宇宙人。金枠SECRETタイル
- **シークレットアイテム**（住人確率+1%でベランダに金色出現）: 金のハト像／小さなUFO／謎の宝箱
- 開発用の全アイテム一覧: `gallery.html`

## 14. サウンド

`js/sound.js`。**Web Audio APIで全音をコード合成**（音声ファイル不使用・依存ゼロ・オフライン）。

- BGM: Am→F→C→G のスニーキーな8bit風ループ（104BPM）。初回タップで開始
- カットイン中は緊迫モードBGMに自動切替（148BPM・半音揺れベース＋アラームパルス＋サイレン）
- 効果音16種: タップ／ヒント出現／強ヒント／噂／警告／発覚／カットイン／帰宅アラーム／タイマーのカチカチ／時間切れ／正解／コンボ／不正解／ゴクゴク／ズーム／ファンファーレ
- 右上ミュートボタン（`localStorage: vt_mute`）

## 16. 品質保証

`node test/validate.js`（プロジェクト直下で実行）で以下を機械チェック（現状オールグリーン）:
- 住人50種・各9ヒント・アイコン/混同idの整合
- マンション10種・部屋20・出現プールのカテゴリ多様性
- 全10マンション×300ターン=3000ターンのシミュレーション（4択整合・二重回答防止・得点/飲み計算・イベント発生）
- 帰宅アンブッシュの飲み対象4パターン・確率反映（設定0で0回/1で全回）
- 待機イベント全8種発生・同一ゲーム内12連続重複なし
- 全ヒントのイラスト分類の安全性・シーン/建物全景SVG生成
- シークレット出現率・通常回への非混入・図鑑永続化
